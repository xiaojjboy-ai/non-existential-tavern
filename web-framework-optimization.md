# 《非存在主义酒馆》Web 框架优化计划 V1

本计划旨在对现有的 Web 开发框架进行视觉表现和交互功能的深度打磨，提升游戏的氛围感和核心玩法体验。

## 1. 客观目标 (Objective)
- **视觉提升**：从色块占位升级为真实的素材渲染，并加入打字机、转场和环境特效。
- **玩法增强**：将调酒环节从简单的文字选项升级为独立的交互式 UI 模块。
- **听觉补完**：实现 BGM 的平滑切换和 SE 音效的精准触发。

## 2. 核心文件与上下文 (Key Files & Context)
- `src/components/GameCanvas.tsx`: PixiJS 渲染核心，将在此实现资源加载和特效。
- `src/app/page.tsx`: UI 表现层，将在此实现打字机效果。
- `src/store/useGameStore.ts`: 状态管理，将在此集成音频控制和调酒逻辑。
- `public/assets/`: 所有的美术和音频资源将存放于此。

## 3. 实施步骤 (Implementation Steps)

### 第一阶段：表现力增强（打字机与资源加载）
1. **实现 Typewriter 组件**：
   - 编写 `src/components/Typewriter.tsx`，支持逐字显示、速度控制和点击立即跳过。
2. **PixiJS 资源加载器**：
   - 在 `GameCanvas.tsx` 中使用 `Assets.load` 实现资源的异步预加载。
   - 支持根据 `resources` 字段加载对应的立绘 (Sprites) 和背景 (BG)。
3. **立绘渲染优化**：
   - 实现立绘的平滑淡入淡出。
   - 增加简单的 Idle 动画（如通过 Y 轴微小正弦波动实现的呼吸感）。

### 第二阶段：氛围感打磨（转场与特效）
1. **转场特效**：
   - 实现 `[BG]` 切换时的 Dissolve（溶解）转场效果，使用 PixiJS 的 `Filter` 或遮罩实现。
2. **环境滤镜**：
   - 为全局 Stage 增加轻微的胶片颗粒感（Film Grain）和色彩偏移（Color Adjust），营造酒馆氛围。
3. **音效管理器集成**：
   - 在 `useGameStore` 中维护当前 BGM 状态。
   - 实现 BGM 切换时的 Cross-fade（淡入淡出）。
   - 实现 `[SE]` 指令的即时触发逻辑。

### 第三阶段：交互式调酒系统
1. **开发 MixingUI 组件**：
   - 当检测到 `[CHOICE drink_XX]` 时，弹出覆盖全屏的调酒交互层。
   - 提供 4-6 种基础酒液的可视化选择。
2. **调酒逻辑闭环**：
   - 玩家选择酒品并点击“提交”后，计算结果并调用 `handleChoice` 返回主剧情流。

## 4. 验证与测试 (Verification & Testing)
- **性能测试**：在低配浏览器中测试 PixiJS 渲染和资源加载是否会导致掉帧。
- **逻辑测试**：验证打字机效果在快速连点时是否会出现文本重叠或逻辑错乱。
- **音效验证**：检查 BGM 切换是否平滑，是否存在爆音或多声道冲突。
- **兼容性验证**：确保调酒 UI 在不同屏幕尺寸下都能正常交互。
