# 对话层模板 — 怎么写台词

## 这层是干什么的？

对话层就是"剧本"。它负责写：

- 角色说了什么话
- 角色做了什么动作（括号里写）
- 酒保说了什么（尽量少说）
- 旁白文字

**对话层里没有背景切换、没有音效、没有选项跳转——只有文字。**

---

## 最基本的写法

```
[dlg_01_01]
Lendro：……还开着？
```

拆开看：
- `[dlg_01_01]` = 这段对话的名字（锚点ID），让指令层能跳到这里
- `Lendro` = 说话的角色名，用中文名
- `：……还开着？` = 角色说的话

---

## 动作描写怎么写？

用中文全角括号 `（）`，写在台词之间，不加角色名前缀：

```
[dlg_01_03]
Lendro：（拉高脚凳，坐下，动作像在审讯室里找位置）
Lendro：嗯。
（风衣扣子解开，帽子摘下放在膝上）
Lendro：有什么能喝的。
```

注意：
- 第一行有角色名，是"他做了这个动作"
- 第三行没有角色名，是"接着发生了这个"，还是同一个角色
- 括号里的内容是给立绘/动画的参考，不是台词

---

## 选项回应怎么写？

选项的文字本身不写在这里（写在数据层）。这里只写**选了之后角色的反应**：

```
[dlg_01_02a]
酒保：开着。坐吧。

[dlg_01_02b]
（酒保擦杯子，不说话）

[dlg_01_02c]
酒保：你淋湿了。
```

每个选项的回应用独立的 `[dlg_锚点]` 节点。锚点命名规则：
- 选项组锚点 + 小写字母后缀
- 比如 `ch_01_01` 的三个选项 → `dlg_01_02a` / `dlg_01_02b` / `dlg_01_02c`

---

## 调酒反应怎么写？

```
[dlg_01_drink_correct]
Lendro：（端起来，没喝，看了看杯中液体的颜色）
Lendro：……清的。
（小口抿了一下）
Lendro：还行。
（微微放松，肩线下降约一指宽）

[dlg_01_drink_wrong_whisky]
Lendro：（端起来闻了一下）
Lendro：……太重了。
（还是喝了一口，放下时杯底磕得比正常响）
Lendro：有清一点的吗。
```

规则：
- 正确酒品 → 角色放松/满意，锚点叫 `dlg_XX_drink_correct`
- 错误酒品 → 角色不拒绝但反应冷淡，锚点叫 `dlg_XX_drink_wrong_{序号}`
- 角色不会因为错误酒品生气，只是冷淡、索要别的、微微失望

---

## 旁白怎么写？

```
[narr_01_01]
吧台上留着一个空杯。
杯底有一圈水渍——分不清是酒，还是雨水。
```

规则：
- 旁白锚点用 `[narr_XX_XX]`
- 出现在角色退场之后
- 风格：短句，意象，留白。不解释，只呈现
- 常见结构：一个场景细节 → 一个隐喻性收束

---

## 酒保怎么写？

**酒保是玩家扮演的角色。核心原则：能不说话就不说话，能短就不长。**

| 情况 | 酒保说什么 |
|------|-----------|
| 角色进门 | 一句话回应，或沉默 |
| 角色讲故事 | 不打断，最多倒酒 |
| 角色问问题 | 最短的回答，或倒酒代替回答 |
| 角色情绪崩溃 | 不说教，不安慰，只倒酒 |
| 告别 | 一句话，或点头 |

好的酒保台词示例：
- `开着。坐吧。`
- `你说，我听。`
- （什么都不说，把酒推过去）

坏的酒保台词示例：
- `我理解你的感受，创伤是需要时间治愈的。` ← 说教了
- `你应该放下过去，向前看。` ← 评判了
- `别担心，一切都会好起来的。` ← 安慰了

---

## 角色声音怎么区分？

每个角色说话方式不同，写台词时要对准：

| 角色 | 说话特点 | 示例 |
|------|---------|------|
| 警探 | 短句，硬朗，克制 | `……还开着？` |
| 赌徒 | 紧张，急促，偶尔炫耀 | `我跟你说，上次在东北角……` |
| 死神 | 优雅，从容，像在陈述事实 | `墙只是在那里。` |
| 猫女（小女孩） | 语焉不详，碎片化 | `……重……不知道……` |
| 猫女（少女） | 多疑，犀利，反问 | `你是那个想带走我的傻子？` |
| 猫女（中年） | 偏执，成熟，有锋芒 | `我只觉得……重。` |
| 牛仔 | 豪爽，嘴硬，粗犷带疲惫 | `我活着就是一切。` |
| 维修师 | 实在，不纠结，偶尔热情 | `啤酒。只有啤酒。` |
| 僵尸 | 极简，不超过5岁水平 | `……我……是谁？` |
| 哲学神 | 平静，像在讲一个很长的故事 | `你知道这个世界是怎么来的吗？` |

---

## 隐喻怎么写？

**隐喻是暗线——草蛇灰线，浅浅一提，不点破。**

写台词时，隐喻埋在表面文字里，不单独标注。标注放在数据层。

好的隐喻台词示例：
- `你见过码头凌晨三点的样子吗？` ← 表面是问天气，暗线是 PTSD 闪回前兆
- `怀表停了上千次时间，却没有一次能停下来让我喘口气。` ← 表面是道具描述，暗线是疲惫

坏的隐喻台词示例：
- `我的创伤后应激障碍让我每天闭眼都看到搭档的惨状。` ← 直接说出来了，不是隐喻
- `就像阿尔茨海默一样，我连后悔都不知道叫什么。` ← 点破了，不是草蛇灰线

---

## 写的时候注意

1. **不要写系统指令**——`[BG]` `[SE]` `[CHOICE]` 属于指令层
2. **不要写数值**——好感度变化属于数据层
3. **每个 `[dlg_锚点]` 必须和指令层的 `[GOTO]` 对应**
4. **括号里的动作描写是给立绘的参考，不是台词**
5. **酒保尽量少说话——倾听比说话更有力量**
