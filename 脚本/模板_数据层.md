# 数据层模板 — 怎么写数值和触发机制

## 这层是干什么的？

数据层就是"游戏规则"。它负责定义：

- 这次来访前，好感度是多少
- 玩家选了什么，好感度怎么变
- 调酒正确/错误有什么后果
- 选项A/B/C分别跳到哪段对话、有什么效果
- 这次的选择会影响后面的哪一夜
- 台词里埋了什么隐喻（给创作者看的，不进游戏）

**数据层里没有台词、没有画面指令——只有规则和数值。**

---

## 数据层用什么格式？

YAML。就是那种用缩进排列的格式，像这样：

```yaml
meta:
  day: 01
  character: Lendro
  visit: 1
```

不用学编程，照着模板填就行。

---

## 完整结构（从上到下）

### 1. meta — 基本信息

```yaml
meta:
  day: 01                    # 第几天，写两位数
  character: Lendro          # 角色英文名
  visit: 1                   # 第几次来访
  requires: null             # 前置条件，没有就写 null
  unlocks:                   # 这次解锁了什么
    - character: Lendro      # 解锁了这个角色
  next: Day02_维修师_第1次   # 下一夜是什么脚本
  resources:                 # 这次需要哪些美术/音效资源
    bg:
      - id: tavern_night_rain
        desc: 酒馆夜间，窗外有雨
    bgm:
      - id: jazz_piano_low
        desc: 低沉爵士钢琴
    se:
      - id: door_open_wind_chime
        desc: 推门，风铃响
    sprite:
      - id: Lendro_coat_wet
        desc: 深灰风衣，肩头水渍
```

**requires 怎么写？**

如果这一夜需要之前发生过什么事才能触发，就写条件：

```yaml
requires:
  - condition: 赌徒和死神都达到Lv.3
    from: Day11_赌徒死神_对峙
```

如果不需要任何前置条件（比如游戏第一天），就写 `null`。

### 2. affinity — 好感度变化

```yaml
affinity:
  character: Lendro          # 哪个角色的好感度
  start: 0                   # 来访前是多少
  max: 5                     # 上限是多少
  changes:                   # 这次可能怎么变
    - id: ch_01_01_C         # 哪个选项触发的
      trigger: "ch_01_01 选C（你淋湿了）"  # 用人话描述
      field: 关切            # 变的是哪个维度
      value: +1              # 变了多少
      note: Day03 更早开口   # 对后面有什么影响
```

**好感度维度有哪些？**

每个角色不一样，在角色设定里定义。常见的：

| 维度 | 含义 | 谁用 |
|------|------|------|
| 信任 | 角色愿意对你敞开多少 | 所有角色 |
| 关切 | 你对角色的关心程度 | 警探、赌徒、僵尸 |
| 亲和 | 角色对你有多亲近 | 警探、牛仔 |
| 理解 | 你对角色困境的理解 | 猫女 |

**怎么理解这些变化？**

- 玩家选了C（"你淋湿了"）→ 关切+1 → 这意味着玩家关心了警探 → 后面Day03警探会更早开口讲深层故事
- 调酒正确（白酒）→ 信任+1 → 警探觉得你懂他
- 选了B（"你不用说了"）→ 信任+1 → 警探觉得你尊重他的边界

### 3. drink — 调酒规则

```yaml
drink:
  id: drink_01_01            # 调酒锚点
  available: [白酒, 威士忌, 啤酒, 柠檬水]  # 酒架上有什么
  correct: 白酒              # 正确答案是什么
  hint: "有什么暖的 / 随便来点什么"         # 角色台词里的暗示
  correct_effect:
    affinity: 信任+1         # 正确了，好感度怎么变
    emotion: 放松            # 角色情绪怎么变
  wrong_effects:
    威士忌:
      dialogue: dlg_01_drink_wrong_whisky   # 跳到哪段对话
      reaction: "嫌太重，要清的"             # 简述反应
    啤酒:
      dialogue: dlg_01_drink_wrong_beer
      reaction: "不喝，要烈的"
    柠檬水:
      dialogue: dlg_01_drink_wrong_lemon
      reaction: "质疑酒架种类"
```

**hint 是什么？**

就是角色在对话层里说的那句话，给玩家暗示该倒什么酒。比如：
- 警探说"有什么暖的" → 暗示白酒
- 赌徒说"讨口水喝" → 暗示柠檬水（没有水）
- 维修师什么都不暗示 → 啤酒永远是啤酒

**没有调酒环节的夜怎么办？**

整段删掉，写 `drink: null`。

### 4. branches — 选项跳转和效果

```yaml
branches:
  ch_01_01:
    A: { goto: dlg_01_02a, effect: 亲和+1 }
    B: { goto: dlg_01_02b, effect: null }
    C: { goto: dlg_01_02c, effect: 关切+1 }
```

**怎么读懂这行？**

- `ch_01_01` = 第1天第1个选项组
- 选A → 跳到对话层 `dlg_01_02a`，好感度亲和+1
- 选B → 跳到对话层 `dlg_01_02b`，好感度不变
- 选C → 跳到对话层 `dlg_01_02c`，好感度关切+1

**effect 是什么？**

就是选了这个选项之后，好感度怎么变。没有变化就写 `null`。

**特殊：同一段对话，不同节奏**

有些选项跳到同一段台词，但节奏不同：

```yaml
ch_01_04:
  A: { goto: dlg_01_08a, effect: null, pace: normal }   # 正常节奏
  B: { goto: dlg_01_08b, effect: null, pace: tight }    # 更紧，角色更紧张
  C: { goto: dlg_01_08c, effect: null, pace: slow }     # 更慢，停顿更长
```

### 5. links — 和其他夜的衔接

```yaml
links:
  next_visit: Day03_警探_第2次       # 这个角色下次什么时候来
  related:                           # 和其他夜的关联
    - day: 03
      character: Lendro
      visit: 2
      conditions:                    # 条件性衔接
        - if: ch_01_05_C             # 如果Day01选了C
          then: 跳过寒暄直接讲 Eilas  # Day03就会发生这个
        - if: ch_01_01_C
          then: 更早开口
    - day: 08
      character: Zombie
      event: 警探拔枪，与Day01摸枪套呼应
```

**怎么理解 conditions？**

就是"如果之前做了某选择，后面就会不一样"。比如：
- Day01你选了"倒酒不问"（C）→ Day03警探会直接讲深层故事，不用你再问
- Day01你选了"你淋湿了"（C）→ Day03警探会更早开口

**event 是什么？**

是无条件的关联——不管你选了什么，这个事件都会发生。比如Day08僵尸进门时警探拔枪，这是设定里的固定事件，和Day01的选择无关。

### 6. metaphor — 隐喻标注

```yaml
metaphor:
  - anchor: dlg_01_07           # 哪段对话里埋了隐喻
    surface: 码头凌晨三点的雾    # 玩家看到的表面意思
    deep: PTSD 闪回前兆         # 暗线真正在说什么
```

**这部分不进游戏逻辑，是给创作者看的。**

写台词时，隐喻已经埋在表面文字里了。这里只是标注一下，方便以后检查：
- 这个隐喻埋得够不够浅（草蛇灰线，不点破）
- 这个隐喻和角色的设定是否一致
- 这个隐喻和其他角色的隐喻是否呼应

## 特殊场景怎么写？

### 双人场景（两个角色同时在场）

```yaml
meta:
  character: [Lendro, Zombie]   # 用列表写两个角色
  visit: null                    # 双人场景没有"第几次来访"

affinity:
  - character: Lendro            # 分别写两个角色的好感度
    start: 2
    max: 5
    changes: [...]
  - character: Zombie
    start: 0
    max: 5
    changes: [...]

drink: null                      # 双人场景通常没有调酒
```

### 结局脚本

```yaml
meta:
  visit: final                   # 用 final 表示这是结局
  is_ending: true                # 标记这是结局
  ending_variants:
    A: 结局A名称                 # 这个角色有几个结局
    B: 结局B名称

branches:
  ch_XX_ending:
    A: { goto: dlg_XX_ending_a, ending: A }
    B: { goto: dlg_XX_ending_b, ending: B }
```

### 第二周目揭示

```yaml
meta:
  day: post                      # 用 post 表示通关后
  character: Robot
  visit: 1
  is_reveal: true                # 标记这是揭示
  requires:
    - condition: 全部角色结局已触发
      from: all

links:
  next_visit: null               # 没有后续了
```

## 写的时候注意

1. **不要写台词**——台词属于对话层
2. **不要写画面指令**——`[BG]` `[SE]` 属于指令层
3. **所有锚点ID必须和指令层、对话层一一对应**
4. **effect 为 null 时可以省略，但建议保留，写清楚"没变化"**
5. **隐喻标注是给创作者看的，不进游戏逻辑**
6. **好感度维度因角色而异，看角色设定文档**
