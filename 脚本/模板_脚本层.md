# 脚本层模板 — 怎么把三层拼成一个关卡文件

## 这层是干什么的？

脚本层就是"成品"。把指令层、对话层、数据层拼在一起，就是一个完整的、可以直接用的关卡文件。

**每一夜 = 一个脚本文件 = 指令层 + 对话层 + 数据层**

---

## 文件命名规则

```
Day{天数两位}_{角色中文名}_第{次数}次.md

示例：
Day01_警探_第1次.md
Day02_维修师_第1次.md
Day08_僵尸_第1次.md
Day09_警探+僵尸_镜像.md      ← 双人场景用加号
Day12_警探_结局.md            ← 结局不加"第X次"
Day19_哲学神_最终.md
DayPost_机器人_揭示.md        ← 通关后用 Post
```

---

## 文件结构

一个脚本文件从上到下，固定三块：

```markdown
# {角色名} — 第{N}次来访（Day XX）

---

# 指令层

（这里放指令层的全部内容）

---

# 对话层

（这里放对话层的全部内容）

---

# 数据层

（这里放数据层的YAML内容）
```

**三块之间用 `---` 分隔，顺序固定：指令层 → 对话层 → 数据层。**

---

## 怎么拼？一步一步来

### 第1步：先写对话层

为什么先写对话？因为对话是核心——角色说了什么、故事怎么推进，决定了其他一切。

1. 按时间顺序写：入场 → 落座 → 调酒 → 核心对话 → 收尾 → 旁白
2. 每段对话标上 `[dlg_锚点]`
3. 每个选项的回应用不同锚点（a/b/c）
4. 写完之后，你就知道有哪些对话节点、哪些选项了

### 第2步：再写指令层

对话层写完之后，你就知道流程了。现在把流程翻译成指令：

1. 开头：场景初始化（背景、音乐、灯光、道具）
2. 入场：音效 + 立绘 + 跳到第一段对话
3. 每个选项组：`[CHOICE]` + 各选项跳转
4. 每段对话之间：`[GOTO]` 跳转
5. 调酒：`[CHOICE]` + 各酒品跳转
6. 退场：立绘消失 + 音效 + 音乐渐弱 + 旁白 + `[END]`

**关键：指令层里每个 `[GOTO]` 和 `[CHOICE]` 里的锚点，必须和对话层的 `[dlg_]` 一一对应。**

### 第3步：最后写数据层

指令层和对话层都写完了，你就知道有多少选项、怎么跳转了。现在填数据：

1. `meta`：天数、角色、前置条件、解锁、资源清单
2. `affinity`：每个选项对好感度的影响
3. `drink`：调酒的正确答案和各错误反应
4. `branches`：把指令层的选项跳转整理成结构化数据，加上 effect
5. `links`：这次的选择会影响后面的哪一夜
6. `metaphor`：台词里埋了什么隐喻

---

## 对应关系检查清单

拼完之后，检查三层的对应关系：

| 检查项 | 怎么查 |
|--------|--------|
| 指令层的每个 `[GOTO xxx]` | 对话层里必须有 `[xxx]` |
| 指令层的每个 `[CHOICE ch_XX_XX]` | 数据层 branches 里必须有 `ch_XX_XX` |
| 对话层的每个 `[dlg_XX_XX]` | 要么被指令层 `[GOTO]` 指向，要么被选项跳转指向 |
| 数据层 branches 的每个 `goto` | 对话层里必须有对应的 `[dlg_]` |
| 数据层 drink 的每个 `dialogue` | 对话层里必须有对应的 `[dlg_]` |
| 数据层 metaphor 的每个 `anchor` | 对话层里必须有对应的 `[dlg_]` 或 `[narr_]` |

**如果发现对不上，说明漏写了或者锚点ID写错了。**

## 完整示例（精简版）

下面是一个最精简的脚本文件，展示三层怎么拼：

```markdown
# 警探 Lendro — 第1次来访（Day 01）

---

# 指令层

## 场景初始化

```
[BG tavern_night_rain]
[BGM jazz_piano_low]
[LIGHT 暖黄偏暗]
[PROP 吧台干净，酒架半满]
```

## 入场

```
[SE door_open_wind_chime]
[ENTER Lendro_coat_wet]
[PAUSE 1.5s]
[GOTO dlg_01_01]
```

## 选项 ch_01_01

```
[CHOICE ch_01_01]
  A → dlg_01_02a
  B → dlg_01_02b
  C → dlg_01_02c
```

## 调酒 drink_01_01

```
[CHOICE drink_01_01]
  白酒   → dlg_01_drink_correct
  威士忌 → dlg_01_drink_wrong_whisky
  啤酒   → dlg_01_drink_wrong_beer
  柠檬水 → dlg_01_drink_wrong_lemon
```

## 退场

```
[EXIT Lendro]
[SE door_close_wind_chime]
[BGM fade_out 3s]
[PAUSE 3s]
[GOTO narr_01_01]
[END]
```

---

# 对话层

## 入场

```
[dlg_01_01]
Lendro：……还开着？
```

### 选项 ch_01_01 回应

```
[dlg_01_02a]
酒保：开着。坐吧。

[dlg_01_02b]
（酒保擦杯子，不说话）

[dlg_01_02c]
酒保：你淋湿了。
```

## 调酒反应

```
[dlg_01_drink_correct]
Lendro：……清的。还行。

[dlg_01_drink_wrong_whisky]
Lendro：……太重了。有清一点的吗。

[dlg_01_drink_wrong_beer]
Lendro：……算了。有烈的吗。

[dlg_01_drink_wrong_lemon]
Lendro：……你这儿就这些？
```

## 旁白

```
[narr_01_01]
吧台上留着一个空杯。
杯底有一圈水渍——分不清是酒，还是雨水。
```

## 写的时候注意

1. **三层顺序固定**：指令层 → 对话层 → 数据层，不要调换
2. **三层用 `---` 分隔**，清晰明了
3. **写之前先写对话层**——故事是核心，其他层为它服务
4. **写完之后做对应检查**——锚点ID必须三层对得上
5. **一个文件 = 一夜**，不要把多夜塞进一个文件
6. **文件名和 `meta.day`、`meta.character`、`meta.visit` 保持一致**
