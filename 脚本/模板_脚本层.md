# 脚本层模板 — 怎么把三层拼成一个关卡文件

## 这层是干什么的？

脚本层就是"成品"。把指令层、对话层、数据层拼在一起，就是一个完整的、可以直接用的关卡文件。

**每一夜 = 一个脚本文件 = 指令层 + 对话层 + 数据层**

---

## 文件命名规则

```
Day{天数两位}_{角色中文名}_第{次数}次.md

示例：
Day01_警探_第1次.md
Day02_维修师_第1次.md
Day08_僵尸_第1次.md
Day09_警探+僵尸_镜像.md      ← 双人场景用加号
Day12_警探_结局.md            ← 结局不加"第X次"
Day19_哲学神_最终.md
DayPost_机器人_揭示.md        ← 通关后用 Post
```

---

## 文件结构

一个脚本文件从上到下，固定三块（注意 V2 标准）：

```markdown
# {角色名} — 第{N}次来访（Day XX）

---

## 指令层

（这里放指令层的全部内容）

---

## 对话层

（这里放对话层的全部内容）

---

## 数据层

（这里放数据层 V2 的YAML内容）
```

**三块之间用 `---` 分隔，顺序固定：指令层 → 对话层 → 数据层。**

---

## 怎么拼？一步一步来

### 第1步：先写对话层

为什么先写对话？因为对话是核心——角色说了什么、故事怎么推进，决定了其他一切。

1. 按时间顺序写：入场 → 落座 → 调酒 → 核心对话 → 收尾 → 旁白
2. 每段对话标上 `[dlg_锚点]`
3. 每个选项的回应用不同锚点（a/b/c）
4. 写完之后，你就知道有哪些对话节点、哪些选项了

### 第2步：再写指令层

对话层写完之后，你就知道流程了。现在把流程翻译成指令：

1. 开头：场景初始化（`[BG bg_id transition]`、`[BGM bgm_id action]`、灯光、道具）
2. 入场：音效 + 立绘 `[ENTER charId pose position]` + 跳到第一段对话
3. 每个选项组：`[CHOICE]` + 各选项跳转
4. 每段对话之间：`[GOTO targetNodeId]` 跳转
5. 调酒：`[CHOICE drink_XX_XX]` + 各 evaluation rule 的跳转
6. 退场：立绘消失 `[EXIT charId]` + 音效 + 音乐渐弱 + 旁白 + `[END]`

**关键：指令层里每个 `[GOTO]` 和 `[CHOICE]` 里的目标，必须和对话层的 `[dlg_]` 一一对应。**

### 第3步：最后写数据层

指令层和对话层都写完了，你就知道有多少选项、怎么跳转了。现在填数据（V2 YAML格式）：

1. `meta`：天数、角色、前置条件、解锁、资源清单
2. `drink`：定义玩家调出什么配方（evaluationRules 条件树）会走向哪个节点，以及加什么好感度。
3. `branches`：把指令层的选项跳转整理成结构化数据，加上 effects
4. `dialogues`：可选，用于给对应的对白节点挂载特殊的音效或配音。
5. `links`：这次的选择会影响后面的哪一夜
6. `metaphor`：台词里埋了什么隐喻

---

## 对应关系检查清单

拼完之后，检查三层的对应关系：

| 检查项 | 怎么查 |
|--------|--------|
| 指令层的每个 `[GOTO xxx]` | 对话层里必须有 `[xxx]` |
| 指令层的每个 `[CHOICE ch_XX_XX]` | 数据层 branches 里必须有 `ch_XX_XX` |
| 数据层 branches 的每个 `gotoNodeId` | 对话层里必须有对应的 `[dlg_]` |
| 数据层 drink 的 evaluationRules `gotoNodeId` | 对话层里必须有对应的 `[dlg_]` |

---

## 完整示例（精简版）

下面是一个最精简的脚本文件，展示三层怎么拼：

```markdown
# 警探 Lendro — 第1次来访（Day 01）

---

## 指令层

### 场景初始化

```
[BG tavern_night_rain none]
[BGM jazz_piano_low play]
[LIGHT #ffeedd 1.0 暖黄偏暗]
[PROP bar show 吧台干净，酒架半满]
```

### 入场

```
[SE door_open_wind_chime]
[ENTER Lendro Lendro_coat_wet center]
[PAUSE 1500]
[GOTO dlg_01_01]
```

### 选项 ch_01_01

```
[CHOICE ch_01_01]
  A → dlg_01_02a
  B → dlg_01_02b
```

### 调酒 drink_01_01

```
[CHOICE drink_01_01]
  drink_01_01_correct → dlg_01_drink_correct
  drink_01_01_wrong → dlg_01_drink_wrong
```

### 退场

```
[EXIT Lendro]
[SE door_close_wind_chime]
[BGM jazz_piano_low fade_out 3000]
[PAUSE 3000]
[GOTO narr_01_01]
[END]
```

---

## 对话层

### 入场

```
[dlg_01_01]
Lendro：……还开着？
```

### 选项 ch_01_01 回应

```
[dlg_01_02a]
酒保：开着。坐吧。

[dlg_01_02b]
（酒保擦杯子，不说话）
```

### 调酒反应

```
[dlg_01_drink_correct]
Lendro：……清的。还行。

[dlg_01_drink_wrong]
Lendro：……太重了。有清一点的吗。
```

### 旁白

```
[narr_01_01]
吧台上留着一个空杯。
杯底有一圈水渍——分不清是酒，还是雨水。
```

---

## 数据层

```yaml
meta:
  day: 01
  character: Lendro
  visit: 1
  requires: null
  unlocks: []
  next: null
  resources: {}

drink:
  id: drink_01_01
  correctRecipe:
    ingredients:
      - id: baijiu
        name: 白酒
        volumeOz: 2
    method: build
    glass:
      type: rock
      iceType: none
  hints: []
  evaluationRules:
    - id: drink_01_01_correct
      match: all
      conditions:
        - field: ingredient.baijiu.volumeOz
          op: gte
          value: 1.5
      gotoNodeId: dlg_01_drink_correct
      affinityEffect:
        character: Lendro
        field: 信任
        value: 1
    - id: drink_01_01_wrong
      match: all
      conditions: []
      gotoNodeId: dlg_01_drink_wrong
      affinityEffect: null

branches:
  ch_01_01:
    A:
      gotoNodeId: dlg_01_02a
      effects:
        - character: Lendro
          field: 亲和
          value: 1
    B:
      gotoNodeId: dlg_01_02b
      effects: []

dialogues:
  dlg_01_01:
    effects:
      soundEffectId: null
      voiceId: lendro_greeting

links:
  related: []
metaphor: []
```
